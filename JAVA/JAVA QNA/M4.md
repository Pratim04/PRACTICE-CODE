### Q1. What is method overloading?

**Answer:**
Method overloading is a feature in Java that allows a class to have **more than one method with the same name**, provided their **parameter lists differ** (in number, type, or order of parameters). It is a form of **compile-time (static) polymorphism**.

```java
class Example {
    void display(int a) { }
    void display(int a, int b) { }   // overloaded
    void display(String s) { }       // overloaded
}
```

---

### Q2. How many types of method overloading are available in Java?

**Answer:**
There are **3 types** of method overloading in Java:

1. **By changing the number of parameters** – e.g., `add(int a, int b)` vs `add(int a, int b, int c)`
2. **By changing the data type of parameters** – e.g., `add(int a, int b)` vs `add(double a, double b)`
3. **By changing the order/sequence of parameters** – e.g., `show(int a, String s)` vs `show(String s, int a)`

---

### Q3. What is type casting?

**Answer:**
Type casting is the **conversion of a variable from one data type to another**. In Java, there are two kinds:

- **Widening (Implicit):** Automatically converting a smaller type to a larger type (e.g., `int` → `double`).
- **Narrowing (Explicit):** Manually converting a larger type to a smaller type using a cast operator (e.g., `double` → `int`).

```java
int a = 10;
double b = a;        // widening (implicit)
int c = (int) 9.99;  // narrowing (explicit) → c = 9
```

### Q5. `"public static void main(String[] args)"` — Why do we write `public static`?

**Answer:**

- **`public`**: The method must be accessible from **outside the class** (by the JVM) without any restrictions. If it were private or protected, the JVM could not call it.
- **`static`**: The JVM needs to call `main()` **without creating an object** of the class. Since static methods belong to the class (not an instance), the JVM can invoke it directly.

---

### Q6. Write down the names of the two types of type casting used in Java.

**Answer:**

1. **Widening Type Casting (Implicit / Automatic Casting)** – converting a smaller data type into a larger one.
   - Example: `int` → `long` → `float` → `double`

2. **Narrowing Type Casting (Explicit / Manual Casting)** – converting a larger data type into a smaller one using a cast operator.
   - Example: `double` → `float` → `long` → `int` → `short` → `byte`

### Q7. What is constructor overloading?

**Answer:**
Constructor overloading is defining **multiple constructors in the same class** with different parameter lists. It allows objects to be initialized in different ways.

```java
class Student {
    Student() { }                          // no-arg constructor
    Student(String name) { }               // one-arg constructor
    Student(String name, int age) { }      // two-arg constructor
}
```

### Q10. What are the advantages of method overloading?

**Answer:**

1. **Increases code readability** – methods with the same name are easier to remember.
2. **Reduces complexity** – no need to use different method names for similar operations.
3. **Promotes reusability** – one logical method name can handle multiple data types or argument counts.
4. **Supports compile-time polymorphism** – the correct method is resolved at compile time.
5. **Improves flexibility** – allows the same method to work with different inputs.

---

### Q11. Can we overload methods at runtime?

**Answer:**
**No.** Method overloading is resolved at **compile time** (static/early binding), not at runtime. The compiler determines which overloaded method to call based on the method signature (number and types of parameters) during compilation itself.

---

### Q12. What is overloading?

**Answer:**
Overloading means defining **multiple methods or constructors with the same name** but with **different signatures** within the same class. Java supports:

- **Method overloading** – same method name, different parameters.
- **Constructor overloading** – same constructor name, different parameters.

It is an example of **compile-time polymorphism**.

---

### Q13. How does the compiler differentiate overloaded methods from duplicate methods?

**Answer:**
The compiler differentiates them by checking the **method signature**, which includes:

1. **Number of parameters**
2. **Data type of parameters**
3. **Order/sequence of parameters**

If two methods have the **same name AND the same parameter list**, they are considered **duplicate (not overloaded)** and the compiler throws a **compile-time error**. Note: **return type alone** does not distinguish overloaded methods.

---

### Q14. Can we overload the `main()` method?

**Answer:**
**Yes**, we can overload the `main()` method. Java allows multiple `main()` methods with different parameter lists. However, only `public static void main(String[] args)` is the **entry point** recognized by the JVM.

```java
public static void main(String[] args) { System.out.println("JVM calls this"); }
public static void main(int a)         { System.out.println("Overloaded main"); }
public static void main(String s)      { System.out.println("Overloaded main"); }
```

### Q15. Why do we use `break` in a switch case?

**Answer:**
`break` is used to **exit the switch block** after a matching case is executed. Without `break`, Java exhibits **fall-through** behavior — execution continues into subsequent cases even if they don't match.

```java
switch (x) {
    case 1: System.out.println("One"); break;  // stops here
    case 2: System.out.println("Two"); break;
}
```

Without `break`, after case 1 matches, case 2 would also execute.

### Q18. Explain the concept of garbage collection in Java.

**Answer:**
**Garbage Collection (GC)** is the process by which the **JVM automatically frees memory** occupied by objects that are no longer reachable or referenced by any part of the program.

**Key Points:**
- Java developers do not manually free memory (unlike C/C++ with `free()`).
- The JVM's **Garbage Collector** runs in the background and reclaims unused heap memory.
- An object becomes eligible for GC when there are **no more references** pointing to it.
- `System.gc()` can *suggest* GC to the JVM, but it is not guaranteed to run immediately.

**How it works:**
1. **Mark:** GC identifies which objects are in use (reachable).
2. **Sweep:** Unreachable objects are removed and memory is freed.
3. **Compact (optional):** Remaining objects are compacted to reduce fragmentation.

**Example:**
```java
Student s = new Student("Alice");
s = null;   // the Student object is now eligible for garbage collection
```

**Benefits:** Prevents memory leaks, simplifies memory management, and improves developer productivity.

### Q19. Why can a top-level Java class not be private?

**Answer:**
A **top-level class cannot be `private`** in Java because:

1. A `private` class would be **accessible only within its own file**, making it completely unusable by any other class or the JVM.
2. The JVM needs to **load and instantiate** top-level classes from outside (e.g., from the command line or other classes); `private` would prevent this.
3. The purpose of `private` is to restrict access within a **class body** — it is a member-level modifier, not a top-level class modifier.

**Valid access modifiers for top-level classes:**
- `public` – accessible from anywhere.
- *(default/package-private)* – accessible within the same package only.

**Note:** Inner (nested) classes *can* be `private` because they are members of an enclosing class.

### Q21. Write a Java program to solve quadratic equations.

**Answer:**
Solves `ax² + bx + c = 0` using the quadratic formula:  
`x = (-b ± √(b² - 4ac)) / 2a`

```java
import java.util.Scanner;

class QuadraticEquation {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        System.out.print("Enter a: ");
        double a = sc.nextDouble();
        System.out.print("Enter b: ");
        double b = sc.nextDouble();
        System.out.print("Enter c: ");
        double c = sc.nextDouble();

        double discriminant = b * b - 4 * a * c;

        if (discriminant > 0) {
            double x1 = (-b + Math.sqrt(discriminant)) / (2 * a);
            double x2 = (-b - Math.sqrt(discriminant)) / (2 * a);
            System.out.println("Two distinct real roots:");
            System.out.println("x1 = " + x1);
            System.out.println("x2 = " + x2);
        } else if (discriminant == 0) {
            double x = -b / (2 * a);
            System.out.println("One repeated real root:");
            System.out.println("x = " + x);
        } else {
            double realPart = -b / (2 * a);
            double imagPart = Math.sqrt(-discriminant) / (2 * a);
            System.out.println("Complex roots:");
            System.out.println("x1 = " + realPart + " + " + imagPart + "i");
            System.out.println("x2 = " + realPart + " - " + imagPart + "i");
        }
    }
}
```

### Q26. How do you use a constructor in Java with an object as a parameter?

**Answer:**
You can pass an **object as a parameter** to a constructor to copy or initialize values from another object (this is called a **copy constructor** pattern).

```java
class Student {
    String name;
    int age;

    // Regular constructor
    Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // Constructor with object as parameter (copy constructor)
    Student(Student s) {
        this.name = s.name;
        this.age  = s.age;
    }

    void display() {
        System.out.println("Name: " + name + ", Age: " + age);
    }

    public static void main(String[] args) {
        Student s1 = new Student("Alice", 20);
        Student s2 = new Student(s1);  // passing object as parameter

        s1.display();
        s2.display();
    }
}
```

**Output:**
```
Name: Alice, Age: 20
Name: Alice, Age: 20
```

```java
class PrimeCheck {
    static boolean isPrime(int n) {
        if (n <= 1) return false;
        if (n == 2) return true;
        if (n % 2 == 0) return false;
        for (int i = 3; i <= Math.sqrt(n); i += 2) {
            if (n % i == 0) return false;
        }
        return true;
    }

    public static void main(String[] args) {
        if (args.length == 0) {
            System.out.println("Usage: java PrimeCheck <number>");
            return;
        }
        int num = Integer.parseInt(args[0]);
        if (isPrime(num))
            System.out.println(num + " is a Prime Number.");
        else
            System.out.println(num + " is NOT a Prime Number.");
    }
}
```

### Q31 / Q38. Design an application using class `AddDistance`.

**Answer:**

```java
class AddDistance {
    private int feet;
    private int inch;

    AddDistance(int feet, int inch) {
        this.feet = feet;
        this.inch = inch;
    }

    AddDistance addDistances(AddDistance d1, AddDistance d2) {
        int totalInch = d1.inch + d2.inch;
        int totalFeet = d1.feet + d2.feet;

        if (totalInch >= 12) {
            totalFeet += totalInch / 12;
            totalInch = totalInch % 12;
        }
        return new AddDistance(totalFeet, totalInch);
    }

    void display() {
        System.out.println("Distance: " + feet + " feet " + inch + " inches");
    }

    public static void main(String[] args) {
        AddDistance d1 = new AddDistance(5, 9);
        AddDistance d2 = new AddDistance(3, 8);

        AddDistance d3 = new AddDistance(0, 0);
        d3 = d3.addDistances(d1, d2);

        System.out.print("Distance 1: "); d1.display();
        System.out.print("Distance 2: "); d2.display();
        System.out.print("Total     : "); d3.display();
    }
}
```

```
Total     : 9 feet 5 inches
```

### Q32 / Q49. Write a Java program to calculate income tax.

```java
class IncomeTax {

            System.out.println("Usage: java IncomeTax <earnings>");
            return;
        }

        double earnings = Double.parseDouble(args[0].replace(",", ""));
        double tax = 0;

        if (earnings <= 50000) {
            tax = 0;
        } else if (earnings <= 60000) {
            tax = (earnings - 50000) * 0.10;
        } else if (earnings <= 150000) {
            tax = (60000 - 50000) * 0.10 + (earnings - 60000) * 0.20;
        } else {
            tax = (60000 - 50000) * 0.10
                + (150000 - 60000) * 0.20
                + (earnings - 150000) * 0.30;
        }

        System.out.println("Income Tax for earnings Rs." + earnings + " is Rs." + tax);
    }
}
```

```java
        this.courses     = new Course[maxCourses];
```

### Q47. Create superclass `Shape` with subclasses `Circle`, `Rectangle`, `Triangle` overriding `area()`.

```java
class Shape {
    double area() { return 0; }
    String name()  { return "Shape"; }
}

class Circle extends Shape {
    double radius;
    Circle(double radius) { this.radius = radius; }

    @Override
    double area() { return Math.PI * radius * radius; }

    @Override
    String name() { return "Circle (r=" + radius + ")"; }
}

class Rectangle extends Shape {
    double length, width;
    Rectangle(double length, double width) {
        this.length = length;
        this.width  = width;
    }

    @Override
    double area() { return length * width; }

    @Override
    String name() { return "Rectangle (" + length + "x" + width + ")"; }
}

class Triangle extends Shape {
    double base, height;
    Triangle(double base, double height) {
        this.base   = base;
        this.height = height;
    }

    @Override
    double area() { return 0.5 * base * height; }

    @Override
    String name() { return "Triangle (b=" + base + ", h=" + height + ")"; }
}

class ShapeDemo {
    public static void main(String[] args) {
        Shape[] shapes = {
            new Circle(7),
            new Rectangle(5, 10),
            new Triangle(6, 8),
            new Circle(3.5),
            new Rectangle(4, 4)
        };

        System.out.println("Areas of Shapes:");
        for (Shape s : shapes)
            System.out.printf("%-35s -> Area = %.2f%n", s.name(), s.area());
    }
}
```
