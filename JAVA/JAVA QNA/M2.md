# Java – Inheritance & Exception Handling: Questions & Answers

---

## Short Answer Questions (2 Marks)

---

**Q1. What is Inheritance in Java?**

Inheritance is an OOP feature in Java that allows one class (subclass/child class) to acquire the properties and methods of another class (superclass/parent class). It promotes **code reusability** and establishes an **"is-a" relationship** between classes.

```java
class Animal {
    void eat() { System.out.println("Animal eats"); }
}

class Dog extends Animal {  // Dog inherits Animal
    void bark() { System.out.println("Dog barks"); }
}
```

The keyword used for inheritance is `extends`.

---

**Q2. What is a superclass and subclass?**

- **Superclass (Parent Class):** The class whose properties and methods are inherited by another class. It is the base or parent class.
- **Subclass (Child Class):** The class that inherits from the superclass. It can add its own fields and methods in addition to those inherited.

```java
class Vehicle {        // Superclass
    int speed = 100;
}

class Car extends Vehicle {  // Subclass
    String brand = "Toyota";
}
```

---

**Q3. Can a class extend more than one class?**

**No.** Java does **not** support multiple inheritance through classes. A class can extend only **one** class at a time. This is to avoid the **Diamond Problem** (ambiguity when two parent classes have the same method).

```java
// Not allowed in Java:
// class C extends A, B { }  // Compilation error

// Multiple inheritance can be achieved through interfaces:
interface A { void show(); }
interface B { void display(); }
class C implements A, B {
    public void show() { }
    public void display() { }
}
```

---

**Q4. Can we extend a final class?**

**No.** A `final` class cannot be extended (subclassed). If you try to extend a final class, the compiler will throw an error. The `String` class in Java is a classic example of a final class.

```java
final class MyClass {
    void greet() { System.out.println("Hello"); }
}

// class SubClass extends MyClass { }  // Compilation Error!
```

---

**Q5. Can a final method be overridden?**

**No.** A `final` method cannot be overridden in a subclass. It can be **inherited** (used in the subclass), but its implementation cannot be changed.

```java
class Parent {
    final void show() {
        System.out.println("Parent's final method");
    }
}

class Child extends Parent {
    // void show() { }  // Compilation Error! Cannot override final method
}
```

---

**Q6. Can we inherit private members of a base class to its subclass?**

**No.** `private` members (fields and methods) of a base class are **not inherited** by the subclass. They are only accessible within the class they are declared in. However, they can be accessed indirectly using `public` or `protected` getter/setter methods.

```java
class Parent {
    private int secret = 42;

    public int getSecret() {  // public getter
        return secret;
    }
}

class Child extends Parent {
    void show() {
        // System.out.println(secret);  // Error! Not accessible
        System.out.println(getSecret());  // OK - indirect access
    }
}
```

---

**Q7. Can we access subclass members if we create an object of a superclass?**

**No.** If an object is created of a superclass type, it can only access the members defined in the superclass. Subclass-specific members are **not accessible** through a superclass reference.

```java
class Animal {
    void eat() { System.out.println("Animal eats"); }
}

class Dog extends Animal {
    void bark() { System.out.println("Dog barks"); }
}

public class Main {
    public static void main(String[] args) {
        Animal a = new Dog();
        a.eat();   // OK
        // a.bark();  // Compilation Error! Animal reference cannot access Dog method
    }
}
```

---

**Q8. Can we override a static method?**

**No.** Static methods **cannot be overridden** in Java. They belong to the class, not to an instance. If a subclass defines a static method with the same signature, it is called **method hiding**, not overriding. It is resolved at **compile time** (not runtime).

```java
class Parent {
    static void show() { System.out.println("Parent static method"); }
}

class Child extends Parent {
    static void show() { System.out.println("Child static method"); }  // Method hiding
}

public class Main {
    public static void main(String[] args) {
        Parent p = new Child();
        p.show();  // Output: Parent static method (not Child's)
    }
}
```

**Q9. What is the difference between error and exception in Java?**

| Feature | Error | Exception |
|---|---|---|
| Definition | Serious problem the program cannot recover from | Condition that the program can handle |
| Package | `java.lang.Error` | `java.lang.Exception` |
| Recoverability | Not recoverable | Recoverable using try-catch |
| Examples | `OutOfMemoryError`, `StackOverflowError` | `NullPointerException`, `IOException` |
| Cause | JVM or system-level failures | Logic errors or external factors |

---

**Q10. What is an exception?**

An **exception** is an unexpected event or condition that disrupts the normal flow of a program during execution. It is an object that represents an error condition and is an instance of the `Throwable` class (specifically `Exception` subclass).

```java
public class Main {
    public static void main(String[] args) {
        int a = 10, b = 0;
        int result = a / b;  // Throws ArithmeticException
    }
}
```

Exceptions can be **checked** (must be handled at compile time) or **unchecked** (occur at runtime).

---

**Q11. What is an error?**

An **error** in Java represents serious problems that occur at the JVM level and are generally beyond the control of the programmer. Errors are not meant to be caught or handled. They are subclasses of `java.lang.Error`.

**Common examples:**
- `OutOfMemoryError` — JVM runs out of heap memory
- `StackOverflowError` — call stack exceeds its limit (e.g., infinite recursion)
- `VirtualMachineError` — JVM is broken

```java
public class Main {
    public static void main(String[] args) {
        main(args);  // Infinite recursion → StackOverflowError
    }
}
```

**Q12. Can we throw an exception manually? If yes, how?**

**Yes.** We can manually throw an exception using the `throw` keyword followed by an instance of an exception class.

**Syntax:**
```java
throw new ExceptionType("message");
```

**Example:**
```java
public class Main {
    static void checkAge(int age) {
        if (age < 18) {
            throw new IllegalArgumentException("Age must be 18 or above.");
        }
        System.out.println("Access granted.");
    }

    public static void main(String[] args) {
        checkAge(15);  // Throws IllegalArgumentException
    }
}
```

**Output:**
```
Exception in thread "main" java.lang.IllegalArgumentException: Age must be 18 or above.
```

**Q13. What are the Exception Handling Keywords in Java?**

Java provides **five** keywords for exception handling:

| Keyword | Purpose |
|---|---|
| `try` | Defines a block of code to monitor for exceptions |
| `catch` | Handles the exception thrown in the try block |
| `finally` | Block that always executes, whether or not an exception occurred |
| `throw` | Used to manually throw an exception |
| `throws` | Declares exceptions that a method might throw |

```java
void method() throws IOException {
    try {
        // risky code
    } catch (IOException e) {
        // handle exception
    } finally {
        // always runs
    }
}
```

**Q14. What is an Abstract class?**

An **abstract class** is a class that cannot be instantiated on its own and may contain one or more abstract methods (methods without a body). It is declared using the `abstract` keyword and serves as a blueprint for its subclasses.

```java
abstract class Shape {
    abstract double area();  // abstract method - no body

    void display() {         // concrete method
        System.out.println("Area: " + area());
    }
}

class Circle extends Shape {
    double radius;
    Circle(double r) { this.radius = r; }

    @Override
    double area() { return Math.PI * radius * radius; }
}
```

**Q15. What is the use of `super` in Java?**

The `super` keyword in Java refers to the **immediate parent class** and is used for three purposes:

1. **Access parent class variables:**
```java
super.variableName;
```

2. **Call parent class methods:**
```java
super.methodName();
```

3. **Call parent class constructor:**
```java
super(arguments);  // must be the first statement in child constructor
```

**Example:**
```java
class Animal {
    String type = "Animal";
    void sound() { System.out.println("Generic sound"); }
}

class Dog extends Animal {
    String type = "Dog";

    void display() {
        System.out.println(super.type);  // Animal
        super.sound();                    // Generic sound
    }
}
```

**Q16. What is the difference between `this()` and `super()` in Java?**

| Feature | `this()` | `super()` |
|---|---|---|
| Refers to | Current class constructor | Parent class constructor |
| Used for | Constructor chaining within the same class | Calling parent class constructor |
| Position | Must be the first statement in the constructor | Must be the first statement in the constructor |
| Default call | Compiler adds `super()` if neither is written | Compiler adds `super()` by default |

```java
class Parent {

}

class Child extends Parent {
    Child() {

        System.out.println("Child constructor");
    }

    Child(int x) {
        this();   // calls Child()
        System.out.println("Child parameterized constructor: " + x);
    }
}
```

**Q19. Can we make abstract methods static in Java?**

**No.** An abstract method cannot be static in Java. This is because:

- `abstract` means the method has no body and must be **overridden** in a subclass.
- `static` methods belong to the class and **cannot be overridden** (only hidden).

These two concepts are contradictory, so the compiler throws an error if you try to combine them.

```java
abstract class Demo {
    // abstract static void show();  // Compilation Error!
}
```

---

**Q20. Can you use `this()` and `super()` both in a constructor?**

**No.** Both `this()` and `super()` must be the **first statement** in a constructor. Since only one statement can be first, you cannot use both in the same constructor. Trying to do so results in a compilation error.

```java
class Parent {
    Parent() { System.out.println("Parent"); }
}

class Child extends Parent {
    Child() {
        super();   // OK
        // this(10);  // Error! Cannot use both
    }
}
```

**Q22. How does multiple inheritance get implemented in Java?**

Java does **not** support multiple inheritance through classes to avoid the **Diamond Problem** (ambiguity when two parent classes have the same method). However, multiple inheritance can be achieved using **interfaces**.

A class can implement multiple interfaces, thereby inheriting the abstract contracts of all of them.

```java
interface Flyable {
    void fly();
}

interface Swimmable {
    void swim();
}

class Duck implements Flyable, Swimmable {
    @Override
    public void fly() {
        System.out.println("Duck is flying.");
    }

    @Override
    public void swim() {
        System.out.println("Duck is swimming.");
    }
}

public class Main {
    public static void main(String[] args) {
        Duck d = new Duck();
        d.fly();
        d.swim();
    }
}
```

**Output:**
```
Duck is flying.
Duck is swimming.
```

Interfaces also support **interface inheritance** (one interface extending multiple interfaces):
```java
interface A { void methodA(); }
interface B { void methodB(); }
interface C extends A, B { }  // C inherits from both A and B
```

**Q23. What are the advantages of using exception handling?**

Exception handling in Java provides several important benefits:

1. **Separation of Error-Handling Code:** Normal code and error-handling code are kept separate using try-catch blocks, improving readability.

2. **Program Continuity:** Instead of crashing, the program can catch exceptions and continue running normally.

3. **Meaningful Error Messages:** Specific exception types allow programmers to display helpful, targeted error messages to users.

4. **Propagation of Errors:** Using `throws`, exceptions can be passed up the call stack to be handled at the appropriate level.

5. **Grouping of Error Types:** Java's exception hierarchy allows catching multiple related exceptions using a single catch block (e.g., catching `Exception` catches all checked exceptions).

6. **Resource Safety:** The `finally` block ensures resources like files and database connections are always closed, preventing memory leaks.

7. **Custom Exceptions:** Programmers can define their own exception classes to handle application-specific errors meaningfully.

**Q24. Write the syntax of a nested try-catch block in Java.**

A **nested try-catch** block is a try block placed inside another try block. It is used when different parts of a code block can throw different types of exceptions.

**Syntax:**
```java
try {
    // Outer try block
    try {
        // Inner try block
        // Code that may throw an exception
    } catch (ExceptionType1 e1) {
        // Handle inner exception
    }
} catch (ExceptionType2 e2) {
    // Handle outer exception
} finally {
    // Always executes
}
```

**Example:**
```java
public class NestedTryCatch {
    public static void main(String[] args) {
        try {
            int[] arr = new int[5];
            try {
                int result = 10 / 0;  // ArithmeticException
            } catch (ArithmeticException e) {
                System.out.println("Inner catch: " + e.getMessage());
            }
            arr[10] = 50;  // ArrayIndexOutOfBoundsException
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("Outer catch: " + e.getMessage());
        } finally {
            System.out.println("Finally block executed.");
        }
    }
}
```

**Output:**
```
Inner catch: / by zero
Outer catch: Index 10 out of bounds for length 5
Finally block executed.
```

**Q25. Discuss different types of inheritance possible in Java.**

Java supports the following types of inheritance:

**1. Single Inheritance:** One class extends one parent class.
```java
class A { }
class B extends A { }
```

**2. Multilevel Inheritance:** A class extends another class which itself extends another class, forming a chain.
```java
class A { }
class B extends A { }
class C extends B { }  // C → B → A
```

**3. Hierarchical Inheritance:** Multiple classes extend the same parent class.
```java
class A { }
class B extends A { }
class C extends A { }
```

**4. Multiple Inheritance (via Interfaces):** A class implements more than one interface. Java does not allow extending multiple classes.
```java
interface A { }
interface B { }
class C implements A, B { }
```

**5. Hybrid Inheritance (via Interfaces):** A combination of two or more types of inheritance, achieved through interfaces.
```java
interface A { }
interface B extends A { }
class C implements B { }
```

> **Note:** Multiple and hybrid inheritance through classes is **not allowed** in Java to avoid the Diamond Problem.

**Q28. Discuss static variable with a proper example.**

A **static variable** (also called a class variable) is shared among all instances of a class. It is declared using the `static` keyword and is created only once in memory when the class is loaded.

**Key characteristics:**
- Shared by all objects of the class.
- Initialized only once.
- Accessed using the class name.
- Useful for storing common data like a counter.

**Example:**
```java
class Student {
    String name;
    static int count = 0;  // static variable - shared by all objects

    Student(String name) {
        this.name = name;
        count++;  // incremented each time an object is created
    }

    void display() {
        System.out.println("Name: " + name + " | Total Students: " + count);
    }
}

public class Main {
    public static void main(String[] args) {
        Student s1 = new Student("Alice");
        Student s2 = new Student("Bob");
        Student s3 = new Student("Carol");

        s1.display();
        s2.display();
        s3.display();
    }
}
```

**Output:**
```
Name: Alice | Total Students: 3
Name: Bob   | Total Students: 3
Name: Carol | Total Students: 3
```

All three objects share the same `count` variable, demonstrating static behavior.

**Q29. Discuss final variable with a proper example.**

A **final variable** is a variable declared with the `final` keyword whose value **cannot be changed** once it is assigned. It behaves like a constant.

**Key characteristics:**
- Must be initialized at declaration or in the constructor.
- Cannot be reassigned after initialization.
- By convention, final variable names are written in UPPER_CASE.

**Example:**
```java
class Circle {
    final double PI = 3.14159;  // final variable
    double radius;

    Circle(double radius) {
        this.radius = radius;
    }

    double area() {
        return PI * radius * radius;
    }
}

public class Main {
    public static void main(String[] args) {
        Circle c = new Circle(7);
        System.out.println("Area: " + c.area());

        // c.PI = 3.14;  // Compilation Error! Cannot reassign final variable
    }
}
```

**Output:**
```
Area: 153.93791
```

**Q30. Show how a reference variable can refer to an object and how a function can be called by that reference variable.**

In Java, an **object** is created in heap memory, and a **reference variable** stores the address (reference) of that object. Methods and fields of the object are accessed via the reference variable using the dot (`.`) operator.

```java
class Person {
    String name;
    int age;

    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    void introduce() {
        System.out.println("Hi, I am " + name + " and I am " + age + " years old.");
    }

    void birthday() {
        age++;
        System.out.println("Happy Birthday, " + name + "! You are now " + age + ".");
    }
}

public class Main {
    public static void main(String[] args) {
        Person p = new Person("Alice", 25);  // p is the reference variable

        p.introduce();    // method called via reference variable
        p.birthday();     // another method call
        p.introduce();
    }
}
```

**Output:**
```
Hi, I am Alice and I am 25 years old.
Happy Birthday, Alice! You are now 26.
Hi, I am Alice and I am 26 years old.
```

Here, `p` is the reference variable that holds the reference to the `Person` object in memory.

**Q32. Explain the `this` keyword with a suitable example.**

The `this` keyword in Java is a reference to the **current object** — the object whose method or constructor is being called. It is used in three main ways:

**1. Resolve naming conflicts between instance variables and parameters:**
```java
class Student {
    String name;
    int age;

    Student(String name, int age) {
        this.name = name;  // 'this.name' = instance variable; 'name' = parameter
        this.age = age;
    }
}
```

**2. Call another constructor in the same class (`this()`):**
```java
class Box {
    int length, width, height;

    Box() {
        this(1, 1, 1);  // calls parameterized constructor
    }

    Box(int l, int w, int h) {
        this.length = l; this.width = w; this.height = h;
    }
}
```

**3. Pass current object as an argument:**
```java
class Printer {
    void print(Student s) {
        System.out.println("Name: " + s.name + ", Age: " + s.age);
    }
}

class Student {
    String name = "Alice";
    int age = 20;

    void showDetails(Printer p) {
        p.print(this);  // passing current object
    }
}
```

**Q33. Explain different types of exception handling with examples.**

Java provides several mechanisms for handling exceptions:

**1. try-catch Block:** Catches and handles a specific exception.
```java
try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("Caught: " + e.getMessage());
}
```

**2. Multiple catch Blocks:** Handles different types of exceptions separately.
```java
try {
    String s = null;
    System.out.println(s.length());
} catch (NullPointerException e) {
    System.out.println("Null Pointer: " + e.getMessage());
} catch (Exception e) {
    System.out.println("General: " + e.getMessage());
}
```

**3. finally Block:** Always executes regardless of exception.
```java
try {
    int x = 5 / 0;
} catch (ArithmeticException e) {
    System.out.println("Exception caught.");
} finally {
    System.out.println("Finally always runs.");
}
```

**4. throw Keyword:** Manually throws an exception.
```java
throw new IllegalArgumentException("Invalid input!");
```

**5. throws Keyword:** Declares exceptions a method may throw.
```java
void method() throws IOException {
    throw new IOException("IO error occurred.");
}
```

**6. Multi-catch (Java 7+):** Catches multiple exceptions in one block.
```java
try {
    // risky code
} catch (ArithmeticException | NullPointerException e) {
    System.out.println("Exception: " + e.getMessage());
}
```

**Q34. Write a program in Java to create a `Box` class with a parameterized constructor with an object argument to initialize `length`, `breadth`, and `height`. Create a function `volume()` that returns the volume.**

```java
class Box {
    double length, breadth, height;

    // Constructor taking a Box object as argument
    Box(Box b) {
        this.length  = b.length;
        this.breadth = b.breadth;
        this.height  = b.height;
    }

    // Parameterized constructor
    Box(double length, double breadth, double height) {
        this.length  = length;
        this.breadth = breadth;
        this.height  = height;
    }

    double volume() {
        return length * breadth * height;
    }

    void display() {
        System.out.println("Length  : " + length);
        System.out.println("Breadth : " + breadth);
        System.out.println("Height  : " + height);
        System.out.println("Volume  : " + volume());
    }
}

public class Main {
    public static void main(String[] args) {
        Box box1 = new Box(5.0, 3.0, 4.0);
        System.out.println("--- Box 1 ---");
        box1.display();

        // Creating box2 using box1 as object argument
        Box box2 = new Box(box1);
        System.out.println("\n--- Box 2 (copy of Box 1) ---");
        box2.display();
    }
}
```

**Output:**
```
--- Box 1 ---
Length  : 5.0
Breadth : 3.0
```

**Q35. Write down the difference between `final`, `finally`, and `finalize` in Java.**

Although these three terms sound similar, they serve entirely different purposes in Java:

| Feature | `final` | `finally` | `finalize()` |
|---|---|---|---|
| Type | Keyword | Keyword | Method |
| Purpose | Restricts modification | Ensures code always executes | Cleanup before GC |
| Applied to | Variables, methods, classes | try-catch blocks | Objects |
| Called by | Programmer | JVM (after try-catch) | Garbage Collector |
| Execution guarantee | Always (compile-time restriction) | Always (even if exception occurs) | Not guaranteed |

**`final` Example:**
```java
final int MAX = 100;
// MAX = 200;  // Error! Cannot reassign a final variable

final class Lock { }
// class Unlock extends Lock { }  // Error! Cannot extend final class
```

**`finally` Example:**
```java
try {
    int result = 10 / 2;
    System.out.println("Result: " + result);
} catch (ArithmeticException e) {
    System.out.println("Exception: " + e.getMessage());
} finally {
    System.out.println("Finally block always executes!");
}
```

**`finalize()` Example:**
```java
class Resource {
    @Override
    protected void finalize() throws Throwable {
        System.out.println("Finalize called – cleaning up resources.");
    }
}

public class Main {
    public static void main(String[] args) {
        Resource r = new Resource();
        r = null;   // eligible for garbage collection
        System.gc(); // request GC
    }
}
```

**Q36. Explain exceptions in Java. Differentiate between checked and unchecked exceptions. Write a program using try-catch to handle `ArithmeticException`.**

**What is an Exception?**

An exception is an abnormal event that occurs during program execution and disrupts the normal flow of instructions. In Java, exceptions are objects of the `Throwable` class hierarchy.

**Checked vs. Unchecked Exceptions:**

| Feature | Checked Exception | Unchecked Exception |
|---|---|---|
| Detection | Detected at **compile time** | Detected at **runtime** |
| Handling | Must be handled or declared | Not mandatory to handle |
| Extends | `Exception` (not `RuntimeException`) | `RuntimeException` |
| Examples | `IOException`, `SQLException` | `ArithmeticException`, `NullPointerException` |

**Program to handle `ArithmeticException`:**
```java
import java.util.Scanner;

            int result = a / b;

            System.out.println("Error: Cannot divide by zero! " + e.getMessage());
```

**Q38. Explain the purpose of the `finally` block. Write a Java program that reads a file, handles `FileNotFoundException`, and ensures file closing using `finally`.**

**Purpose of `finally` Block:**

The `finally` block in Java is used to execute **important cleanup code** that must run regardless of whether an exception occurred or not. It is typically used to:
- Close file streams or database connections
- Release system resources
- Log messages or status information

Even if a `return` statement is encountered in the try or catch block, the `finally` block still executes.

```java
import java.io.FileReader;
import java.io.FileNotFoundException;
import java.io.IOException;

public class FileReadDemo {
    public static void main(String[] args) {
        FileReader fr = null;

        try {
            fr = new FileReader("data.txt");
            System.out.println("File opened successfully.");

            int character;
            while ((character = fr.read()) != -1) {
                System.out.print((char) character);
            }

        } catch (FileNotFoundException e) {
            System.out.println("Error: File not found! " + e.getMessage());

        } catch (IOException e) {
            System.out.println("Error reading file: " + e.getMessage());

        } finally {
            // This block always runs — ensures the file is always closed
            System.out.println("\n[Finally] Closing file resources...");
            if (fr != null) {
                try {
                    fr.close();
                    System.out.println("[Finally] File closed successfully.");
                } catch (IOException e) {
                    System.out.println("[Finally] Error closing file: " + e.getMessage());
                }
            }
        }
    }
}
```

**Output (if file does not exist):**
```
Error: File not found! data.txt (No such file or directory)
[Finally] Closing file resources...
```

```java
class InsufficientBalanceException extends Exception {
    private double shortfall;

    }

        return shortfall;
    }
}

// BankAccount Class
class BankAccount {
    private String owner;
    private double balance;

    BankAccount(String owner, double initialBalance) {
        this.owner = owner;
        this.balance = initialBalance;
    }

    void deposit(double amount) {
        balance += amount;
        System.out.println("Deposited Rs." + amount + " | New Balance: Rs." + balance);
    }

    void withdraw(double amount) throws InsufficientBalanceException {
        if (amount > balance) {
            double shortfall = amount - balance;
            throw new InsufficientBalanceException(shortfall);
        }
        balance -= amount;
        System.out.println("Withdrawn Rs." + amount + " | Remaining Balance: Rs." + balance);
    }

    void displayBalance() {
        System.out.println(owner + "'s Balance: Rs." + balance);
    }
}

            System.out.println("Transaction Failed: " + e.getMessage());
            System.out.println("Shortfall Amount: Rs." + e.getShortfall());
        }

        account.displayBalance();
    }
}
```

**Q41. Discuss best practices for exception handling in Java. Write a program simulating an online shopping cart handling invalid quantity, out-of-stock, and payment failures.**

**Best Practices for Exception Handling:**

1. **Catch specific exceptions** — avoid catching generic `Exception` unless necessary.
2. **Never swallow exceptions** — always log or handle them meaningfully.
3. **Use custom exceptions** — create descriptive exception classes for domain-specific errors.
4. **Always close resources** — use `finally` or try-with-resources.
5. **Don't use exceptions for flow control** — exceptions are for exceptional situations, not regular logic.
6. **Provide meaningful messages** — include context in exception messages.
7. **Log exceptions properly** — use logging frameworks instead of `System.out.println`.
8. **Document exceptions** — use `@throws` Javadoc tag for declared exceptions.

**Program: Online Shopping Cart Simulation**

```java
// Custom Exceptions
class InvalidQuantityException extends Exception {
    InvalidQuantityException(String msg) { super(msg); }
}

class OutOfStockException extends Exception {
    OutOfStockException(String msg) { super(msg); }
}

class PaymentFailureException extends Exception {
    PaymentFailureException(String msg) { super(msg); }
}

// Product Class
class Product {
    String name;
    double price;
    int stock;

    Product(String name, double price, int stock) {
        this.name  = name;
        this.price = price;
        this.stock = stock;
    }
}

// Shopping Cart
class ShoppingCart {

    void addToCart(Product product, int quantity)
            throws InvalidQuantityException, OutOfStockException {

        if (quantity <= 0) {
            throw new InvalidQuantityException(
                "Quantity must be greater than zero. Entered: " + quantity);
        }

        if (quantity > product.stock) {
            throw new OutOfStockException(
                "Only " + product.stock + " unit(s) of '" + product.name +
                "' available. Requested: " + quantity);
        }

        product.stock -= quantity;
        System.out.println("Added to cart: " + quantity + " x " + product.name +
                           " | Subtotal: Rs." + (quantity * product.price));
    }

    void processPayment(double cartTotal, double amountPaid)
            throws PaymentFailureException {

        if (amountPaid < cartTotal) {
            throw new PaymentFailureException(
                "Payment failed! Required Rs." + cartTotal +
                " but received Rs." + amountPaid);
        }

        System.out.println("Payment successful! Total Paid: Rs." + amountPaid);
        double change = amountPaid - cartTotal;
        if (change > 0) {
            System.out.println("Change returned: Rs." + change);
        }
    }
}

// Main Class
public class OnlineShoppingDemo {
    public static void main(String[] args) {
        ShoppingCart cart  = new ShoppingCart();
        Product laptop     = new Product("Laptop",  55000, 3);
        Product headphones = new Product("Headphones", 1500, 2);
        double totalCost   = 0;

        // Test 1: Invalid Quantity
        System.out.println("=== Test 1: Invalid Quantity ===");
        try {
            cart.addToCart(laptop, -1);
        } catch (InvalidQuantityException e) {
            System.out.println("Error: " + e.getMessage());
        } catch (OutOfStockException e) {
            System.out.println("Error: " + e.getMessage());
        }

        // Test 2: Valid Purchase
        System.out.println("\n=== Test 2: Valid Purchase ===");
        try {
            cart.addToCart(laptop, 2);
            totalCost += 2 * laptop.price;
            cart.addToCart(headphones, 1);
            totalCost += 1 * headphones.price;
        } catch (InvalidQuantityException | OutOfStockException e) {
            System.out.println("Error: " + e.getMessage());
        }

        // Test 3: Out of Stock
        System.out.println("\n=== Test 3: Out of Stock ===");
        try {
            cart.addToCart(headphones, 5);  // Only 1 left
        } catch (InvalidQuantityException e) {
            System.out.println("Error: " + e.getMessage());
        } catch (OutOfStockException e) {
            System.out.println("Error: " + e.getMessage());
        }

        // Test 4: Payment Failure
        System.out.println("\n=== Test 4: Payment Failure ===");
        try {
            cart.processPayment(totalCost, 50000);  // Not enough
        } catch (PaymentFailureException e) {
            System.out.println("Error: " + e.getMessage());
        }

        // Test 5: Successful Payment
        System.out.println("\n=== Test 5: Successful Payment ===");
        try {
            System.out.println("Cart Total: Rs." + totalCost);
            cart.processPayment(totalCost, 115000);
        } catch (PaymentFailureException e) {
            System.out.println("Error: " + e.getMessage());
        }
    }
}
```
