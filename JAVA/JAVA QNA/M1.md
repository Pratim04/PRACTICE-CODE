**Q1. Explain byte code.**

Bytecode is the intermediate, platform-independent code generated by the Java compiler when a `.java` source file is compiled. It is stored in `.class` files and is not machine code — instead, it is executed by the Java Virtual Machine (JVM). This is what makes Java platform-independent: the same bytecode can run on any device that has a JVM installed.

---

**Q2. Explain final variable.**

A `final` variable in Java is a constant whose value cannot be changed once it has been assigned. It must be initialized either at the time of declaration or in the constructor.

```java
final int MAX = 100; // value cannot be reassigned
```

---

**Q3. What is a final class?**

A `final` class is a class that cannot be subclassed (extended). It is used to prevent inheritance, typically for security or design reasons. `String` is a well-known example of a final class in Java.

```java
final class MyClass {
    // cannot be extended
}
```

---

**Q4. Define final method.**

A `final` method is a method that cannot be overridden by any subclass. It is used when the implementation of a method must remain unchanged across all subclasses.

```java
class Parent {
    final void display() {
        System.out.println("This cannot be overridden.");
    }
}
```

---

**Q5. What is a constructor in Java?**

A constructor is a special method in Java that is automatically called when an object of a class is created. It has the same name as the class and has no return type. Its main purpose is to initialize the object's attributes.

```java
class Car {
    String brand;
    Car(String b) {  // constructor
        brand = b;
    }
}
```

---

**Q6. Why is Java called portable?**

Java is called portable because Java programs are compiled into bytecode, which is platform-independent. This bytecode can be executed on any operating system or hardware that has a JVM installed, following the principle of **"Write Once, Run Anywhere (WORA)"**.

**Q7. What is `new` in Java used to create an object?**

The `new` keyword in Java is used to dynamically allocate memory for an object at runtime and to invoke the constructor of the class. It returns a reference to the newly created object.

```java
Car c = new Car("Toyota"); // 'new' creates the object
```

---

**Q8. What is a default constructor?**

A default constructor is a constructor with no parameters. If a programmer does not define any constructor in a class, the Java compiler automatically provides a default constructor that initializes all instance variables to their default values (e.g., `0` for int, `null` for objects).

```java
class Dog {
    Dog() {  // default constructor
        System.out.println("Dog created");
    }
}
```

**Q9. What is a command line argument?**

Command line arguments are values passed to a Java program at the time of execution from the command prompt or terminal. They are received as a `String[]` array in the `main()` method parameter `args`.

```bash
java MyProgram Hello World
```
Here, `"Hello"` and `"World"` are command line arguments accessible as `args[0]` and `args[1]`.

**Q10. What is auto widening and explicit narrowing?**

- **Auto Widening (Implicit Type Casting):** Java automatically converts a smaller data type to a larger one without any explicit instruction. No data is lost.
  ```java
  int a = 10;
  double b = a; // int automatically widened to double
  ```

- **Explicit Narrowing (Explicit Type Casting):** Converting a larger data type to a smaller one must be done manually using a cast operator. Data loss may occur.
  ```java
  double x = 9.99;
  int y = (int) x; // explicit narrowing → y = 9
  ```

**Q13. Explain why OOP is better compared to procedural programming languages.**

Object-Oriented Programming (OOP) offers several advantages over procedural programming:

1. **Modularity:** Code is organized into classes and objects, making it easier to manage and maintain large programs.

2. **Reusability:** Through inheritance, existing code can be reused in new classes, reducing redundancy.

3. **Data Hiding (Encapsulation):** Data is wrapped inside classes and access is controlled using access specifiers (`private`, `public`), improving security.

4. **Polymorphism:** The same method name can behave differently depending on the object, making the code more flexible and extensible.

5. **Easier Maintenance:** Since functionality is divided into objects, locating and fixing bugs is easier.

6. **Real-World Modeling:** OOP concepts mirror real-world entities, making design more intuitive.

In contrast, procedural programming follows a top-down approach with functions and shared global data, making large programs harder to manage and debug.

**Q14. Explain how a String object can be created by string literals.**

In Java, a `String` object can be created using a **string literal**, which is a sequence of characters enclosed in double quotes.

```java
String name = "Java";
```

When this statement is executed:
- Java checks the **String Constant Pool** (a special memory area in the heap).
- If the string `"Java"` already exists in the pool, the reference is reused.
- If it does not exist, a new `String` object is created in the pool.

**Advantages of using string literals:**
- Memory efficient — identical strings share the same pool object.
- Faster creation compared to using `new String()`.

**Comparison with `new` keyword:**
```java
String s1 = "Hello";           // stored in String pool
String s2 = new String("Hello"); // stored in heap memory (new object always)
```

Using literals is preferred because they are memory-efficient and avoid unnecessary object creation.

**Q15. Discuss and differentiate between `==` and the `equals()` method.**

| Feature | `==` Operator | `equals()` Method |
|---|---|---|
| Type | Operator | Method defined in `Object` class |
| Compares | **References** (memory addresses) | **Content** (actual values) |
| Use with primitives | Compares values directly | Not applicable |
| Use with objects | Checks if both refer to the same object | Checks if the content of the objects is the same |

**Example:**
```java
String s1 = new String("Hello");
String s2 = new String("Hello");

System.out.println(s1 == s2);       // false (different objects in memory)
System.out.println(s1.equals(s2));  // true  (same content)
```

**With string literals:**
```java
String s3 = "Hello";
String s4 = "Hello";

System.out.println(s3 == s4);       // true (same pool reference)
System.out.println(s3.equals(s4));  // true (same content)
```

**Conclusion:** Use `==` to check reference equality and `equals()` to check value/content equality.

**Q16. *"Constructor in Java can be either public or default"* — Justify.**

This statement is **true**, and can be justified as follows:

**Public Constructor:**
- A `public` constructor can be accessed from any class, in any package.
- It is used when objects of the class need to be created from outside the class or package.
```java
public class Person {
    public Person() {  // public constructor
        System.out.println("Person created");
    }
}
```

**Default Constructor:**
- If no constructor is defined, Java provides a default (no-argument) constructor automatically with package-level access.
- It can also be explicitly declared without an access modifier, giving it package-private access.
```java
class Student {
    Student() {  // default (package-private) constructor
        System.out.println("Student created");
    }
}
```

**Other Access Modifiers:**
- Constructors can also be `private` (used in Singleton pattern) or `protected` (used in inheritance), but the most common forms are `public` and the implicit default.

**Conclusion:** Constructors in Java are flexible in their access control, and the most commonly used access levels are `public` (accessible everywhere) and default (accessible within the same package), justifying the given statement.

**Q17. Develop two different classes to perform withdraw, deposit, and balance check functions. Create a main class to call functions to perform operations. Give the user the facility to choose the operation.**

```java
import java.util.Scanner;

class BankAccount {
    private double balance;

    BankAccount(double initialBalance) {
        this.balance = initialBalance;
    }

    void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            System.out.println("Deposited: " + amount);
        } else {
            System.out.println("Invalid deposit amount.");
        }
    }

    void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            System.out.println("Withdrawn: " + amount);
        } else {
            System.out.println("Insufficient balance or invalid amount.");
        }
    }

    void checkBalance() {
        System.out.println("Current Balance: " + balance);
    }
}

class BankOperations {
    void performOperation(BankAccount account, int choice, double amount) {
        switch (choice) {
            case 1: account.deposit(amount); break;
            case 2: account.withdraw(amount); break;
            case 3: account.checkBalance(); break;
            default: System.out.println("Invalid choice.");
        }
    }
}

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        BankAccount account = new BankAccount(1000);
        BankOperations ops = new BankOperations();
        int choice;

        do {
            System.out.println("\n1. Deposit\n2. Withdraw\n3. Check Balance\n4. Exit");
            System.out.print("Enter choice: ");
            choice = sc.nextInt();
            if (choice != 4) {
                double amount = 0;
                if (choice == 1 || choice == 2) {
                    System.out.print("Enter amount: ");
                    amount = sc.nextDouble();
                }
                ops.performOperation(account, choice, amount);
            }
        } while (choice != 4);

        System.out.println("Thank you for using our bank.");
        sc.close();
    }
}
```

**Q18. Explain the `finalize()` method with its proper syntax. How is it different from a destructor in C++?**

**`finalize()` Method in Java:**

The `finalize()` method is defined in the `java.lang.Object` class and is called by the Garbage Collector just before an object is reclaimed from memory. It allows the object to perform cleanup operations (e.g., closing a file or releasing resources) before being destroyed.

**Syntax:**
```java
protected void finalize() throws Throwable {
    // cleanup code
    System.out.println("Object is being garbage collected");
}
```

**Example:**
```java
class Demo {
    protected void finalize() throws Throwable {
        System.out.println("Finalize called before GC");
    }

    public static void main(String[] args) {
        Demo d = new Demo();
        d = null; // eligible for GC
        System.gc(); // request GC
    }
}
```

**Differences between `finalize()` in Java and Destructor in C++:**

| Feature | `finalize()` in Java | Destructor in C++ |
|---|---|---|
| Called by | Garbage Collector (JVM) | Automatically when object goes out of scope |
| Predictability | Not guaranteed — may never be called | Always called deterministically |
| Syntax | `protected void finalize()` | `~ClassName()` |
| Memory Management | Handled by JVM | Manual (programmer's responsibility) |
| Overriding | Can be overridden | Can be overridden (virtual destructor) |
| Performance | May cause GC delays | Immediate cleanup |

**Q20. Develop a Java program to read a String from the user through the command prompt. Then in that string, replace each vowel with its next consonant.**

```java
public class ReplaceVowels {
    public static boolean isVowel(char c) {
        return "aeiouAEIOU".indexOf(c) != -1;
    }

    public static char nextConsonant(char c) {
        char next = (char)(c + 1);
        while (isVowel(next)) {
            next = (char)(next + 1);
        }
        return next;
    }

    public static void main(String[] args) {
        if (args.length == 0) {
            System.out.println("Please provide a string as a command line argument.");
            return;
        }

        String input = args[0];
        StringBuilder result = new StringBuilder();

        for (int i = 0; i < input.length(); i++) {
            char c = input.charAt(i);
            if (isVowel(c)) {
                result.append(nextConsonant(c));
            } else {
                result.append(c);
            }
        }

        System.out.println("Original String : " + input);
        System.out.println("Modified String : " + result);
    }
}
```

**Run:**
```
java ReplaceVowels "hello"
```
**Output:**
```
Original String : hello
Modified String : hfllp
```

**Q29. Explain the concept of inheritance in Java with an example. Create a base class `Animal` with a method `makeSound()` and two derived classes `Dog` and `Cat` that override the method. Demonstrate polymorphism.**

**Inheritance** is an OOP feature that allows one class (subclass) to acquire the properties and methods of another class (superclass), enabling code reuse and establishing an "is-a" relationship.

```java
class Animal {
    String name;

    Animal(String name) {
        this.name = name;
    }

    void makeSound() {
        System.out.println(name + " makes a sound.");
    }
}

class Dog extends Animal {
    Dog(String name) {
        super(name);
    }

    @Override
    void makeSound() {
        System.out.println(name + " says: Woof!");
    }
}

class Cat extends Animal {
    Cat(String name) {
        super(name);
    }

    @Override
    void makeSound() {
        System.out.println(name + " says: Meow!");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal a1 = new Dog("Rex");   // polymorphism
        Animal a2 = new Cat("Whiskers");

        a1.makeSound();
        a2.makeSound();
    }
}
```

**Output:**
```
Rex says: Woof!
Whiskers says: Meow!
```

**Q37. Create `Person`, `Student`, and `Teacher` subclasses. Find average age using inheritance.**

```java
class Person {
    String name;
    int age;

    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

class Student extends Person {
    String course;

    Student(String name, int age, String course) {
        super(name, age);
        this.course = course;
    }
}

class Teacher extends Person {
    String subject;

    Teacher(String name, int age, String subject) {
        super(name, age);
        this.subject = subject;
    }
}

public class Main {
    public static void main(String[] args) {
        Student[] students = {
            new Student("Alice", 20, "CS"),
            new Student("Bob", 22, "IT"),
            new Student("Carol", 21, "ECE")
        };

        Teacher[] teachers = {
            new Teacher("Mr. Smith", 40, "Math"),
            new Teacher("Ms. Jones", 35, "Science")
        };

        int studentAgeSum = 0;
        for (Student s : students) studentAgeSum += s.age;

        int teacherAgeSum = 0;
        for (Teacher t : teachers) teacherAgeSum += t.age;

        System.out.println("Average Student Age: " + (studentAgeSum / students.length));
        System.out.println("Average Teacher Age: " + (teacherAgeSum / teachers.length));
    }
}
```

```
Average Teacher Age: 37
```

**Q38. Demonstrate dynamic method dispatch using `Shape`, `Circle`, and `Square`. Call overridden `area()` at runtime.**

**Dynamic Method Dispatch** is the mechanism by which a call to an overridden method is resolved at runtime rather than compile time, enabling runtime polymorphism.

```java
abstract class Shape {
    abstract double area();

    void displayArea() {
        System.out.println("Area: " + area());
    }
}

class Circle extends Shape {
    double radius;

    Circle(double radius) {
        this.radius = radius;
    }

    @Override
    double area() {
        return Math.PI * radius * radius;
    }
}

class Square extends Shape {
    double side;

    Square(double side) {
        this.side = side;
    }

    @Override
    double area() {
        return side * side;
    }
}

public class Main {
    public static void main(String[] args) {
        Shape[] shapes = {
            new Circle(5),
            new Square(4),
            new Circle(3)
        };

        for (Shape s : shapes) {
            System.out.print(s.getClass().getSimpleName() + " → ");
            s.displayArea();
        }
    }
}
```

**Output:**
```
Circle → Area: 78.53981633974483
Square → Area: 16.0
Circle → Area: 28.274333882308138
```
