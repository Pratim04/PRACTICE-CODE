## Short Answer Questions (2 Marks)

---

**Q1. What is an interface?**

An **interface** in Java is a blueprint of a class that contains only abstract methods (until Java 7) and constants. From Java 8 onwards, interfaces can also have `default` and `static` methods. It is declared using the `interface` keyword and is used to achieve **full abstraction** and **multiple inheritance** in Java.

```java
interface Printable {
    void print();  // abstract method
}
```

---

**Q2. Can we have methods in an interface?**

**Yes.** An interface can have the following types of methods:

| Version | Allowed Methods |
|---|---|
| Java 7 and below | Only `abstract` methods (no body) |
| Java 8+ | `abstract`, `default` (with body), and `static` (with body) |
| Java 9+ | `private` methods also allowed |

```java
interface Demo {
    void abstractMethod();                        // abstract

    default void defaultMethod() {                // default (Java 8+)
        System.out.println("Default method");
    }

    static void staticMethod() {                  // static (Java 8+)
        System.out.println("Static method");
    }
}
```

---

**Q3. What type of variable is possible in an interface?**

Only **`public static final`** variables (constants) are allowed in an interface. Even if you don't write these modifiers explicitly, the compiler adds them automatically. This means all variables in an interface are constants and **cannot be changed**.

```java
interface Config {
    int MAX_SIZE = 100;       // implicitly public static final
    String APP_NAME = "MyApp"; // implicitly public static final
}
```

---

**Q4. Is it possible to have methods with a definition in an interface?**

**Yes**, but only from **Java 8 onwards**. Two types of methods with bodies are allowed:

- **`default` methods** — provide a default implementation that implementing classes can optionally override.
- **`static` methods** — utility methods that belong to the interface itself.

```java
interface Greeter {
    default void greet() {           // method with definition
        System.out.println("Hello!");
    }

    static void info() {             // static method with definition
        System.out.println("This is a Greeter interface.");
    }
}
```

In Java 7 and before, all interface methods had to be abstract (no body).

---

**Q5. Which keyword is required to use an interface in a class?**

The **`implements`** keyword is used by a class to use (implement) an interface.

```java
interface Flyable {
    void fly();
}

class Bird implements Flyable {      // 'implements' keyword
    @Override
    public void fly() {
        System.out.println("Bird is flying.");
    }
}
```

> **Note:** The keyword `extends` is used when an **interface inherits another interface**.

---

**Q6. How many interfaces can be used by a class at a time?**

A class can implement **any number of interfaces** at a time (multiple interface implementation is fully supported in Java). This is how Java achieves multiple inheritance.

```java
interface A { void methodA(); }
interface B { void methodB(); }
interface C { void methodC(); }

class MyClass implements A, B, C {
    public void methodA() { System.out.println("A"); }
    public void methodB() { System.out.println("B"); }
    public void methodC() { System.out.println("C"); }
}
```

---

**Q7. Is it possible to have inheritance between interfaces?**

**Yes.** An interface can extend one or more other interfaces using the `extends` keyword. This is called **interface inheritance**.

```java
interface Animal {
    void breathe();
}

interface Pet extends Animal {       // interface extending interface
    void play();
}

interface ServiceAnimal extends Animal, Pet {  // extending multiple interfaces
    void work();
}
```

A class implementing the child interface must provide implementations for all methods from the parent interfaces as well.

**Q8. What is a package in Java?**

A **package** in Java is a namespace that groups related classes, interfaces, and sub-packages together. It helps in organizing code, avoiding naming conflicts, and controlling access.

```java
package com.myapp.utilities;  // package declaration

public class MathUtils {
    // class code
}
```

Packages are of two types:
- **Built-in packages** — provided by Java (e.g., `java.lang`, `java.util`)
- **User-defined packages** — created by the programmer

**Q10. Write the declaration syntax of packages.**

A package is declared using the `package` keyword as the **very first statement** in a Java source file (before any imports or class declarations).

**Syntax:**
```java
package packagename;
```

**Example:**
```java
package com.company.project;

public class MyClass {
    // class body
}
```

For sub-packages, use dot notation:
```java
package com.company.project.module;
```

**Q11. Which keyword is required to use a package?**

The **`import`** keyword is used to bring classes or entire packages into the current Java file so they can be used without their fully qualified name.

**Syntax:**
```java
import packagename.ClassName;     // import a specific class
import packagename.*;             // import all classes in the package
```

**Example:**
```java
import java.util.Scanner;         // import specific class

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
    }
}
```

> **Note:** `java.lang` package is automatically imported — no explicit `import` needed.

---

**Q12. What is a sub-package?**

A **sub-package** is a package that is defined inside another package. It is used to further organize related classes into a hierarchical structure, represented using dot notation.

```java
// Main package
package com.myapp;

// Sub-package
package com.myapp.database;

// Another sub-package
package com.myapp.database.queries;
```

**Example import:**
```java
import com.myapp.database.DBConnection;
```

Sub-packages are **independent** — importing a parent package does NOT automatically import its sub-packages.

---

**Q13. Are there any access specifiers for packages?**

Java does not have a direct access specifier specifically for packages. However, the **default (package-private)** access level acts as an implicit access control for packages. When no access modifier is specified, the member is accessible only within the **same package**.

| Access Modifier | Same Class | Same Package | Subclass | Other Package |
|---|---|---|---|---|
| `public` | ✅ | ✅ | ✅ | ✅ |
| `protected` | ✅ | ✅ | ✅ | ❌ |
| Default (none) | ✅ | ✅ | ❌ | ❌ |
| `private` | ✅ | ❌ | ❌ | ❌ |

**Q14. What is a thread?**

A **thread** is the smallest unit of execution within a process. It is a lightweight sub-process that runs independently and shares the same memory space as other threads within the same program. Java supports multithreading natively through the `Thread` class and `Runnable` interface.

```java
class MyThread extends Thread {
    public void run() {
        System.out.println("Thread running: " + Thread.currentThread().getName());
    }
}
```

---

**Q15. What is a process?**

A **process** is an independent program in execution that has its own memory space, resources, and system state. Each process runs in isolation from other processes. A process can contain multiple threads that share the same memory space.

| Feature | Process | Thread |
|---|---|---|
| Memory | Own separate memory | Shares process memory |
| Creation | Heavy (more overhead) | Lightweight |
| Communication | Complex (IPC needed) | Easy (shared memory) |
| Example | Running two separate Java programs | Two threads inside one Java program |

---

**Q16. What is multi-threaded programming?**

**Multi-threaded programming** is a programming technique where multiple threads execute concurrently within a single program, enabling tasks to run simultaneously. It is used to:

- Improve CPU utilization
- Increase application responsiveness
- Perform background tasks (e.g., file download while UI remains active)

```java
class Task extends Thread {
    String name;
    Task(String name) { this.name = name; }

    public void run() {
        for (int i = 1; i <= 3; i++) {
            System.out.println(name + " - Step " + i);
        }
    }
}

public class Main {
    public static void main(String[] args) {
        new Task("Thread-A").start();
        new Task("Thread-B").start();
    }
}
```

---

**Q17. What is the main thread?**

The **main thread** is the thread that is automatically created by the JVM when a Java program starts. It is the first thread to begin execution and is responsible for running the `main()` method. All other (child) threads are created from the main thread.

```java
public class MainThreadDemo {
    public static void main(String[] args) {
        Thread t = Thread.currentThread();  // get the main thread
        System.out.println("Current Thread: " + t.getName());
        System.out.println("Priority: " + t.getPriority());
    }
}
```

**Output:**
```
Current Thread: main
Priority: 5
```

---

**Q18. Which interface is required to implement a child thread?**

The **`Runnable`** interface (from `java.lang`) is used to create a child thread. It has a single abstract method `run()` that contains the code to be executed by the thread.

```java
class MyTask implements Runnable {
    @Override
    public void run() {
        System.out.println("Child thread running: " + Thread.currentThread().getName());
    }
}

public class Main {
    public static void main(String[] args) {
        Thread t = new Thread(new MyTask());  // passing Runnable to Thread
        t.start();
    }
}
```

> Alternatively, a class can extend the `Thread` class and override its `run()` method.

---

**Q19. How many child threads are possible in a single Java program?**

There is **no fixed limit** defined by the Java language specification on the number of child threads in a program. The practical limit depends on:

- The **operating system** and its thread scheduling capacity
- The **available memory** (each thread consumes stack memory)
- **JVM settings** (e.g., `-Xss` for stack size per thread)

Typically, thousands of threads can be created, but for scalable applications, thread pools (via `ExecutorService`) are recommended to manage resources efficiently.

---

**Q20. What is the default thread priority of threads in Java?**

The **default thread priority** in Java is **5** (`Thread.NORM_PRIORITY`). Thread priorities range from **1** (lowest) to **10** (highest).

| Constant | Value | Meaning |
|---|---|---|
| `Thread.MIN_PRIORITY` | 1 | Lowest priority |
| `Thread.NORM_PRIORITY` | 5 | Default priority |
| `Thread.MAX_PRIORITY` | 10 | Highest priority |

```java
Thread t = new Thread();
System.out.println(t.getPriority());  // Output: 5
t.setPriority(Thread.MAX_PRIORITY);   // Set to 10
```

**Q21. Discuss the importance of interfaces in Java.**

Interfaces are one of the most powerful features of Java and play a crucial role for several reasons:

**1. Achieving Full Abstraction:**
Interfaces hide implementation details and expose only what a class can do, achieving 100% abstraction.

**2. Multiple Inheritance:**
Java does not support multiple class inheritance, but a class can implement multiple interfaces, enabling multiple inheritance of type.

**3. Loose Coupling:**
Code written against interfaces is loosely coupled. Implementations can change without affecting client code.

**4. Defining Contracts:**
Interfaces act as contracts — any class implementing an interface is guaranteed to provide all its methods.

**5. Polymorphism:**
Interface references can point to any implementing class object, enabling runtime polymorphism.

**6. Plug-and-Play Architecture:**
New implementations can be swapped in without modifying existing code (e.g., JDBC drivers).

```java
interface Shape {
    double area();
}

class Circle implements Shape {
    double r;
    Circle(double r) { this.r = r; }
    public double area() { return Math.PI * r * r; }
}

class Rectangle implements Shape {
    double l, w;
    Rectangle(double l, double w) { this.l = l; this.w = w; }
    public double area() { return l * w; }
}

public class Main {
    static void printArea(Shape s) {        // works with any Shape
        System.out.println("Area: " + s.area());
    }

    public static void main(String[] args) {
        printArea(new Circle(5));
        printArea(new Rectangle(4, 6));
    }
}
```

**Q22. *"Interface provides full abstraction"* — Justify.**

**Full abstraction** means hiding all implementation details and showing only the functionality. An interface achieves this completely because:

1. **No implementation in the interface:** All methods (traditionally) have no body — only declarations. The class that implements the interface provides the actual logic.

2. **No direct instantiation:** You cannot create an object of an interface, so the internal structure is completely hidden.

3. **Only behavior is exposed:** The interface defines *what* to do, not *how* to do it.

4. **Variables are constants:** All variables are `public static final`, so no mutable state is exposed.

```java
interface ATM {
    void insertCard();
    void enterPin(int pin);
    void withdrawMoney(double amount);
    void ejectCard();
}

class SBIAtm implements ATM {
    public void insertCard()            { System.out.println("SBI: Card inserted."); }
    public void enterPin(int pin)       { System.out.println("SBI: PIN verified."); }
    public void withdrawMoney(double a) { System.out.println("SBI: Rs." + a + " dispensed."); }
    public void ejectCard()             { System.out.println("SBI: Card ejected."); }
}
```

The user of the `ATM` interface sees only the operations — the internal banking logic (PIN verification, balance checks, etc.) is completely hidden. This demonstrates **full abstraction**.

**Q23. Write and discuss the syntax of interface declaration.**

**Syntax:**
```java
[access_modifier] interface InterfaceName [extends OtherInterface] {

    // Constants (implicitly public static final)
    dataType CONSTANT_NAME = value;

    // Abstract methods (implicitly public abstract)
    returnType methodName(parameters);

    // Default method (Java 8+)
    default returnType methodName(parameters) {
        // body
    }

    // Static method (Java 8+)
    static returnType methodName(parameters) {
        // body
    }
}
```

**Discussion:**
- The `interface` keyword is used to declare it.
- All variables are implicitly `public static final`.
- All methods (without `default`/`static`) are implicitly `public abstract`.
- An interface can extend one or more interfaces using `extends`.
- Interfaces cannot have constructors or instance fields.

**Complete Example:**
```java
interface Vehicle {
    int MAX_SPEED = 200;               // public static final

    void accelerate();                 // public abstract
    void brake();                      // public abstract

    default void honk() {             // default method
        System.out.println("Beep beep!");
    }

    static void info() {              // static method
        System.out.println("Vehicle Interface v1.0");
    }
}

class Car implements Vehicle {
    public void accelerate() { System.out.println("Car accelerating..."); }
    public void brake()      { System.out.println("Car braking..."); }
}
```

**Q24. Explain the differences between an abstract class and an interface.**

| Feature | Abstract Class | Interface |
|---|---|---|
| Keyword | `abstract class` | `interface` |
| Inheritance | Single (`extends`) | Multiple (`implements`) |
| Methods | Abstract + concrete | Abstract, `default`, `static` |
| Variables | Any type (instance, static) | Only `public static final` |
| Constructors | Allowed | Not allowed |
| Access modifiers | Any | Methods are `public` by default |
| Speed | Slightly faster | Slightly slower (due to indirection) |
| Use case | Closely related classes sharing code | Unrelated classes sharing a contract |

**When to use which:**
- Use **abstract class** when classes share common code/state (e.g., `Animal` with `eat()`).
- Use **interface** when unrelated classes need to follow the same contract (e.g., `Comparable`, `Serializable`).

```java
abstract class Animal {
    String name;
    Animal(String name) { this.name = name; }
    abstract void makeSound();
    void breathe() { System.out.println(name + " breathes."); }
}

interface Trainable {
    void performTrick();
}

class Dog extends Animal implements Trainable {
    Dog(String name) { super(name); }
    public void makeSound()    { System.out.println(name + ": Woof!"); }
    public void performTrick() { System.out.println(name + " sits."); }
}
```

**Q25. Discuss the significance of partial implementation of an interface.**

When a class **does not implement all methods** of an interface, it must be declared **abstract**. This is called **partial implementation**. The remaining unimplemented methods are left to concrete subclasses.

**Significance:**
1. Allows **incremental implementation** — useful when some methods are generic and others are specific.
2. Enables creation of **abstract base classes** that provide common behavior while deferring specific behavior.
3. Promotes the **Template Method pattern** in design.

```java
interface Animal {
    void eat();
    void sleep();
    void makeSound();
}

// Partial implementation - abstract class implements only some methods
abstract class Mammal implements Animal {
    public void eat()   { System.out.println("Mammal is eating."); }
    public void sleep() { System.out.println("Mammal is sleeping."); }
    // makeSound() is NOT implemented - left to subclass
}

// Concrete class provides the remaining implementation
class Dog extends Mammal {
    public void makeSound() { System.out.println("Dog says: Woof!"); }
}

public class Main {
    public static void main(String[] args) {
        Dog d = new Dog();
        d.eat();
        d.sleep();
        d.makeSound();
    }
}
```

**Output:**
```
Mammal is eating.
Mammal is sleeping.
Dog says: Woof!
```

**Q26. Write and explain the syntax of inheritance of interfaces.**

An interface can inherit from one or more interfaces using the `extends` keyword. Unlike class inheritance, **interface inheritance supports multiple inheritance**.

**Syntax:**
```java
interface ChildInterface extends ParentInterface1, ParentInterface2 {
    // additional methods
}
```

**Example:**
```java
interface Printable {
    void print();
}

interface Showable {
    void show();
}

// Interface inheriting multiple interfaces
interface DisplayUnit extends Printable, Showable {
    void display();
}

// Class implementing the child interface must implement ALL methods
class Monitor implements DisplayUnit {
    public void print()   { System.out.println("Printing..."); }
    public void show()    { System.out.println("Showing..."); }
    public void display() { System.out.println("Displaying on monitor..."); }
}

public class Main {
    public static void main(String[] args) {
        Monitor m = new Monitor();
        m.print();
        m.show();
        m.display();
    }
}
```

**Output:**
```
Printing...
Showing...
Displaying on monitor...
```

A class implementing `DisplayUnit` must implement all three methods — `print()`, `show()`, and `display()`.

**Q27. Discuss the importance of packages in a Java program.**

Packages are a fundamental organizational feature in Java and are important for several reasons:

**1. Organization:** Related classes and interfaces are grouped together, making the codebase easier to navigate and maintain.

**2. Naming Collision Avoidance:** Two classes with the same name can coexist in different packages. For example, `java.util.Date` and `java.sql.Date` are distinct.

**3. Access Control:** Packages work with access modifiers to control visibility. Default (package-private) members are only accessible within the same package.

**4. Reusability:** Packages allow code to be easily shared and reused across different projects.

**5. Modularity:** Large programs are broken into smaller, independently manageable modules via packages.

**6. Built-in Library Organization:** Java's entire standard library is organized into packages (`java.lang`, `java.util`, `java.io`, etc.).

```java
package com.school.management;

public class Student {
    private String name;
    private int roll;

    public Student(String name, int roll) {
        this.name = name;
        this.roll = roll;
    }

    public void display() {
        System.out.println("Roll: " + roll + ", Name: " + name);
    }
}
```

**Q28. `import packagename.*;` — Discuss the advantages and disadvantages.**

The wildcard import `import packagename.*;` imports **all public classes** from a specified package into the current file.

**Advantages:**

| Benefit | Explanation |
|---|---|
| Convenience | No need to write individual import statements for each class |
| Shorter code | Reduces the number of lines in large files using many classes |
| Easier refactoring | Adding a new class from the same package requires no new import |

**Disadvantages:**

| Drawback | Explanation |
|---|---|
| Naming conflicts | If two packages have classes with the same name, ambiguity arises |
| Reduced readability | It's not clear which specific classes are being used |
| Unnecessary loading (logical) | Makes it harder to track dependencies |
| IDE confusion | IDEs may struggle to auto-resolve class origins |

**Example of naming conflict:**
```java
import java.util.*;   // contains Date
import java.sql.*;    // also contains Date

// Compiler is confused — which Date?
Date d = new Date();  // Compilation error: ambiguous
```

**Best practice:** Use specific imports for clarity:
```java
import java.util.ArrayList;
import java.util.HashMap;
```

**Q29. Explain how naming collision can be avoided by packages.**

A **naming collision** occurs when two classes in the same program have the same name, causing ambiguity. Java packages solve this using **namespaces** — a fully qualified class name includes the package path, making it unique.

**Without packages (collision):**
```java
// Two classes both named "Date" — impossible to use both!
class Date { }  // your custom Date
// java.util.Date also exists — conflict!
```

**With packages (resolved):**
```java
java.util.Date utilDate = new java.util.Date();  // Java's Date
java.sql.Date sqlDate = new java.sql.Date(0);    // SQL Date
```

**Practical example:**
```java
import java.util.Date;       // explicitly import one
// use fully qualified name for the other
java.sql.Date sqlDate = new java.sql.Date(System.currentTimeMillis());
Date utilDate = new Date();  // refers to java.util.Date
```

**Company naming convention** to prevent global conflicts:
```
com.google.maps.Location
com.amazon.aws.S3Client
com.mycompany.project.UserService
```

Using the reversed domain name as the package prefix ensures global uniqueness across all libraries.

**Q30. Explain the importance of the main thread in Java multi-threaded programming.**

The **main thread** is the entry point of every Java program and holds a central role in multi-threaded programming:

**1. Program Entry Point:** The JVM creates the main thread automatically and begins execution from the `main()` method.

**2. Parent of All Threads:** All child threads are spawned from the main thread. If the main thread finishes before child threads, the JVM continues until all non-daemon threads complete.

**3. Controls Thread Lifecycle:** The main thread can start, pause (`join()`), or stop other threads.

**4. Access to Thread API:** The main thread can use `Thread.currentThread()` to get its own reference and modify its name or priority.

**5. Program Termination:** The JVM terminates when all non-daemon threads (including the main thread) have finished execution.

```java
public class MainThreadDemo {
    public static void main(String[] args) throws InterruptedException {
        Thread main = Thread.currentThread();
        System.out.println("Main thread: " + main.getName());

        Thread child = new Thread(() -> {
            for (int i = 1; i <= 3; i++) {
                System.out.println("Child thread step: " + i);
            }
        });

        child.start();
        child.join();  // main thread waits for child to finish

        System.out.println("Main thread finished.");
    }
}
```

**Output:**
```
Main thread: main
Child thread step: 1
Child thread step: 2
Child thread step: 3
Main thread finished.
```

**Q31. Explain the syntax and calling mechanism of the `sleep()` method of the `Thread` class.**

The `sleep()` method pauses the execution of the **currently running thread** for a specified duration (in milliseconds). It is a **static method** of the `Thread` class.

**Syntax:**
```java
Thread.sleep(long milliseconds);                         // basic
Thread.sleep(long milliseconds, int nanoseconds);        // with nanoseconds
```

**Key Points:**
- It is a `static` method — always affects the currently executing thread.
- It throws **`InterruptedException`** (checked exception), so it must be handled.
- The thread enters the **TIMED_WAITING** state during sleep.
- After the sleep duration, the thread returns to the **RUNNABLE** state.

**Example:**
```java
public class SleepDemo {
    public static void main(String[] args) {
        System.out.println("Thread starting...");

        for (int i = 1; i <= 5; i++) {
            System.out.println("Count: " + i);
            try {
                Thread.sleep(1000);  // pause for 1 second
            } catch (InterruptedException e) {
                System.out.println("Thread interrupted: " + e.getMessage());
            }
        }

        System.out.println("Thread finished.");
    }
}
```

**Output (one line per second):**
```
Thread starting...
Count: 1
Count: 2
Count: 3
Count: 4
Count: 5
Thread finished.
```

**Q38. Write a program to calculate the area of a square and circle using method overloading.**

**Method overloading** allows multiple methods with the same name but different parameter lists in the same class. The compiler decides which method to call based on the arguments.

```java
class AreaCalculator {

    // Calculate area of a square
    double area(double side) {
        return side * side;
    }

    // Calculate area of a circle
    double area(double radius, boolean isCircle) {
        return Math.PI * radius * radius;
    }

    // Calculate area of a rectangle (bonus overload)
    double area(double length, double width) {
        return length * width;
    }
}

public class Main {
    public static void main(String[] args) {
        AreaCalculator calc = new AreaCalculator();

        double squareSide  = 6.0;
        double circleRadius = 5.0;

        System.out.println("=== Area Calculator ===");
        System.out.printf("Square  (side = %.1f)    → Area = %.2f%n",
                          squareSide, calc.area(squareSide));
        System.out.printf("Circle  (radius = %.1f)  → Area = %.2f%n",
                          circleRadius, calc.area(circleRadius, true));
        System.out.printf("Rectangle (4.0 x 7.0)    → Area = %.2f%n",
                          calc.area(4.0, 7.0));
    }
}
```

**Output:**
```
=== Area Calculator ===
Square  (side = 6.0)    → Area = 36.00
Circle  (radius = 5.0)  → Area = 78.54
Rectangle (4.0 x 7.0)   → Area = 28.00
```

**Q39. Explain the life cycle of a Java applet. Write a Java program to create a simple applet that displays your name and changes the background color. Describe how `init()`, `start()`, `stop()`, and `destroy()` work.**

**Java Applet Life Cycle:**

A Java applet passes through five stages managed by the browser/applet viewer:

```
Born (init) → Running (start) → Idle (stop) → Dead (destroy)
                  ↑                  ↓
                  └──────────────────┘
               (user returns to page / leaves page)
```

| Method | When Called | Purpose |
|---|---|---|
| `init()` | Once, when applet is first loaded | Initialize variables, UI setup |
| `start()` | Every time the page is visited/revisited | Start/resume execution |
| `stop()` | Every time user leaves the page | Pause execution |
| `destroy()` | Once, when applet is permanently removed | Cleanup resources |
| `paint()` | Whenever the applet needs to redraw | Render output on screen |

**Applet Program:**
```java
import java.applet.Applet;
import java.awt.*;

/*
 * HTML to run this applet:
 * <applet code="MyApplet.class" width="400" height="200"></applet>
 */
public class MyApplet extends Applet {

    String message;
    Font titleFont;

    @Override
    public void init() {
        // Called once when applet loads
        setBackground(Color.CYAN);         // change background color
        message = "Hello! My name is Alice.";
        titleFont = new Font("Arial", Font.BOLD, 22);
        System.out.println("init() called - Applet initialized.");
    }

    @Override
    public void start() {
        // Called every time the applet becomes active
        System.out.println("start() called - Applet started.");
    }

    @Override
    public void paint(Graphics g) {
        // Renders content on the applet window
        g.setFont(titleFont);
        g.setColor(Color.DARK_GRAY);
        g.drawString(message, 50, 80);
        g.setFont(new Font("Arial", Font.PLAIN, 16));
        g.setColor(Color.BLUE);
        g.drawString("Welcome to Java Applets!", 80, 120);
    }

    @Override
    public void stop() {
        // Called when user leaves the page
        System.out.println("stop() called - Applet paused.");
    }

    @Override
    public void destroy() {
        // Called once when applet is permanently closed
        System.out.println("destroy() called - Applet destroyed.");
    }
}
```

**Life Cycle in Action:**
1. User opens page → `init()` then `start()` → `paint()` renders content
2. User switches tab → `stop()` is called
3. User returns → `start()` is called again → `paint()` re-renders
4. Browser closes → `stop()` then `destroy()` is called

> **Note:** Java Applets are deprecated since Java 9 and removed in Java 17. Modern alternatives include JavaFX or web technologies.

**Q40 & Q42. Write a Java program to read contents of a text file using `FileInputStream` and write them to another file using `FileOutputStream`. Handle exceptions. Explain byte streams vs character streams.**

**Byte Streams vs. Character Streams:**

| Feature | Byte Streams | Character Streams |
|---|---|---|
| Base classes | `InputStream`, `OutputStream` | `Reader`, `Writer` |
| Data unit | 8-bit bytes | 16-bit Unicode characters |
| Best for | Binary files (images, audio) | Text files |
| Example classes | `FileInputStream`, `FileOutputStream` | `FileReader`, `FileWriter` |
| Encoding | No automatic encoding handling | Handles character encoding automatically |

**Program:**
```java
import java.io.FileInputStream;
import java.io.FileOutputStream;
import java.io.FileNotFoundException;
import java.io.IOException;

public class FileCopyDemo {
    public static void main(String[] args) {
        String sourceFile = "source.txt";
        String destFile   = "destination.txt";

        FileInputStream  fis = null;
        FileOutputStream fos = null;

        try {
            // Open source file for reading (byte stream)
            fis = new FileInputStream(sourceFile);

            // Open destination file for writing (byte stream)
            fos = new FileOutputStream(destFile);

            System.out.println("Copying from '" + sourceFile + "' to '" + destFile + "'...");

            int byteData;
            int totalBytes = 0;

            // Read one byte at a time until end of file (-1)
            while ((byteData = fis.read()) != -1) {
                fos.write(byteData);   // write byte to destination
                totalBytes++;
            }

            System.out.println("File copied successfully! Total bytes: " + totalBytes);

        } catch (FileNotFoundException e) {
            System.out.println("Error: File not found - " + e.getMessage());

        } catch (IOException e) {
            System.out.println("Error during file operation - " + e.getMessage());

        } finally {
            // Always close streams in finally block
            System.out.println("\n[Finally] Closing file streams...");

            if (fis != null) {
                try {
                    fis.close();
                    System.out.println("[Finally] Source file stream closed.");
                } catch (IOException e) {
                    System.out.println("[Finally] Error closing input stream.");
                }
            }

            if (fos != null) {
                try {
                    fos.close();
                    System.out.println("[Finally] Destination file stream closed.");
                } catch (IOException e) {
                    System.out.println("[Finally] Error closing output stream.");
                }
            }
        }
    }
}
```

**Output (if source.txt exists):**
```
Copying from 'source.txt' to 'destination.txt'...
File copied successfully! Total bytes: 42

[Finally] Closing file streams...
[Finally] Source file stream closed.
[Finally] Destination file stream closed.
```

**Output (if source.txt is missing):**
```
Error: File not found - source.txt (No such file or directory)

[Finally] Closing file streams...
```

**Q41. Write a Java program to serialize and deserialize a `Student` object. Explain why serialization is important.**

**What is Serialization?**

**Serialization** is the process of converting a Java object into a byte stream so it can be saved to a file or transmitted over a network. **Deserialization** is the reverse — reconstructing the object from the byte stream.

**Why is Serialization Important?**

| Reason | Explanation |
|---|---|
| Persistence | Save object state to a file and restore it later |
| Network transmission | Send objects between systems over a network |
| Deep copy | Create a complete independent copy of an object |
| Caching | Store computed objects for faster access later |
| Session management | Used in web applications to save session data |

A class must implement the `java.io.Serializable` interface (a marker interface) to be serializable.

**Program:**
```java
import java.io.*;

// Student class implementing Serializable
class Student implements Serializable {
    private static final long serialVersionUID = 1L;  // version control

    String name;
    int rollNumber;
    double marks;

    Student(String name, int rollNumber, double marks) {
        this.name       = name;
        this.rollNumber = rollNumber;
        this.marks      = marks;
    }

    void display() {
        System.out.println("Name       : " + name);
        System.out.println("Roll Number: " + rollNumber);
        System.out.println("Marks      : " + marks);
    }
}

public class SerializationDemo {
    static final String FILE = "student.ser";

    // Serialize: save Student object to file
    static void serialize(Student s) {
        try (ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream(FILE))) {
            oos.writeObject(s);
            System.out.println("Student object serialized and saved to '" + FILE + "'.");
        } catch (IOException e) {
            System.out.println("Serialization error: " + e.getMessage());
        }
    }

    // Deserialize: read Student object from file
    static Student deserialize() {
        Student s = null;
        try (ObjectInputStream ois = new ObjectInputStream(new FileInputStream(FILE))) {
            s = (Student) ois.readObject();
            System.out.println("Student object deserialized from '" + FILE + "'.");
        } catch (FileNotFoundException e) {
            System.out.println("File not found: " + e.getMessage());
        } catch (IOException | ClassNotFoundException e) {
            System.out.println("Deserialization error: " + e.getMessage());
        }
        return s;
    }

    public static void main(String[] args) {
        // Create Student object
        Student original = new Student("Alice", 101, 92.5);

        System.out.println("=== Original Student ===");
        original.display();

        System.out.println("\n--- Serializing ---");
        serialize(original);

        System.out.println("\n--- Deserializing ---");
        Student restored = deserialize();

        if (restored != null) {
            System.out.println("\n=== Restored Student ===");
            restored.display();

            System.out.println("\nSame data? " +
                (original.name.equals(restored.name) &&
                 original.rollNumber == restored.rollNumber &&
                 original.marks == restored.marks));
        }
    }
}
```

**Output:**
```
=== Original Student ===
Name       : Alice
Roll Number: 101
Marks      : 92.5

--- Serializing ---
Student object serialized and saved to 'student.ser'.

--- Deserializing ---
Student object deserialized from 'student.ser'.

=== Restored Student ===
Name       : Alice
Roll Number: 101
Marks      : 92.5

Same data? true
```

> **Note:** Use `transient` keyword on a field to exclude it from serialization (e.g., `transient String password;`).
